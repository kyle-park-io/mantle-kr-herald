# Mantle KR 번역 스타일 가이드

> 초안 (2026-07-14, 웹조사 + 0xMantleKR 실제 용례 기반). KR 팀이 계속 다듬습니다.
> `glossary.json`(용어 규칙)·`locale.json`(표기 규칙)과 함께 번역 프롬프트의 공유 컨텍스트로 쓰입니다.

## 1. 어조 (Tone)

- **존댓말(합니다체)** 을 기본으로 합니다. 공지·발표는 정중하고 명료하게.
- 과한 격식/번역투는 지양하고, 크립토 커뮤니티가 자연스럽게 읽는 문장으로.
- 홍보성 과장·수식은 절제. 사실 중심으로 전달하되 Mantle의 자신감 있는 톤은 유지.

## 2. 음차 vs 번역 vs 원문 유지

- 세부 규칙은 `glossary.json`을 **최우선**으로 따릅니다.
- 일반 원칙:
  - **제품·프로토콜·브랜드명, 티커, 약어** → 원문 유지 (Mantle Network, mETH, MNT, MI4, FBTC, L2, TVL, DA)
  - **정착된 크립토 기술 용어** → 음차 (메인넷, 스테이킹, 리스테이킹, 롤업, 에어드랍, 디파이, 거버넌스, 온체인)
  - **일반 명사로 자연스러운 것** → 번역 (지갑, 검증인, 데이터 가용성)
  - 브랜드 "Mantle"은 러닝 텍스트에서 **"맨틀"** 로 음차 (0xMantleKR 용례). 단 복합 제품명은 원문 유지.

## 3. 약어·티커

- 티커는 원문 대문자 유지: MNT, mETH, cmETH, FBTC. 금액 문맥에선 `$MNT` 형태 허용.
- 약어는 원문 유지하되, 처음 등장 시 필요하면 풀이 병기: "TVL(예치총액)", "DA(데이터 가용성)".

## 4. 문장·표기

- 영어 관용구를 직역하지 말고 의미 중심으로 자연스럽게. (예: "under the hood" → "내부적으로")
- 능동태 우선. 불필요한 피동·번역투("~되어집니다") 지양.
- 한 문장이 길면 끊어서. 트윗은 간결하게.

## 5. 소셜 요소 (트윗/텔레그램 원문에 있을 때)

- **해시태그**: 고유명사 해시태그(#Mantle, #mETH)는 원문 유지. 일반 해시태그는 맥락에 맞게 한글화 가능.
- **멘션(@handle)**: 원문 유지.
- **이모지**: 원문 뉘앙스를 해치지 않는 선에서 보존.
- **링크**: 그대로 유지.
- **스레드 구분자**: 원문에서 트윗과 트윗 사이를 나누는 `---`(수평선)은 **번역에서도 같은 위치에 그대로 유지**합니다. 각 트윗을 개별로 번역하고 사이의 `---`를 보존하세요(트윗 경계가 사라지지 않도록).

## 6. 로케일

- 날짜·숫자·통화·단위 표기는 `locale.json`을 따릅니다 (한국 표기 규칙).

## 7. 검수 포인트

- 용어집 규칙 위반 여부, 오역, 어색한 번역투, 톤 일관성, 티커/약어 표기, 링크·멘션·해시태그 보존.
