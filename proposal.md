# KR 팀 소셜미디어 자동화 제안서

## 0. 들어가며

- **1차 기능 개발은 Kyle의 로컬 컴퓨터에서 동작하는 것을 전제**로 합니다. KR 팀은 클라우드(Google 또는 Lark 드라이브)에 업로드되는 결과물을 통해 진행 상황을 확인할 수 있습니다.
- 1차 기능 개발이 완료되고 KR 팀의 AI 사용 현황이 파악되면, 각자의 로컬 컴퓨터에서 직접 사용할 수 있도록 열어드리겠습니다.
- 자동화 툴은 **보안·안전을 위해 각자의 로컬 컴퓨터에서만 실행**하는 것을 원칙으로 합니다.

## 2. 전체 흐름

```
영어 데이터 수집(API) → 한글 번역 → 1차 드라이브 업로드(Google/Lark) → Lark 알림
   → [1차 검수] → 아이템별 글 변환 → 소셜미디어별 포매팅 → [2차 검수] → 소셜미디어 업로드
```

전체 과정은 **사람이 직접 진행하는 검수**와 **자동으로 처리되는 단계**로 명확히 구분하여 설계하였습니다.

| 구분 | 단계                    |          처리 주체           |
| ---- | ----------------------- | :--------------------------: |
| 1    | 데이터 수집 / 한글 번역 |          🤖 자동화           |
| 2    | **1차 검수**            |           👤 사람            |
| 3    | 아이템별 글 변환        |          🤖 자동화           |
| 4    | 소셜미디어별 포매팅     |          🤖 자동화           |
| 5    | **2차 검수**            |           👤 사람            |
| 6    | 소셜미디어 업로드       | 🤖 자동화 / 👤 매뉴얼 (선택) |

> 검수 이후의 작업(업로드 등)을 자동화로 진행할지 매뉴얼로 진행할지는 자유로이 선택할 수 있습니다.

## 3. 한글 번역

- 한글 번역은 **각자 로컬에서 사용하는 AI Agent**를 통해 진행합니다. (추후 범용적으로 발전 시 Claude API로 전환 가능)
- 현재는 **Kyle의 Claude(Opus 4.8)** 로 진행합니다.
- 번역이 완료된 글은 Google 또는 Lark 드라이브로 업로드됩니다.

### 프롬프트 구성 — 6가지 요소

| 요소                                      | 내용                                                                                      |
| ----------------------------------------- | ----------------------------------------------------------------------------------------- |
| **① 역할 지정** (Role / System Prompt)    | AI에게 역할과 맥락을 부여합니다. 예) "크립토 씬을 잘 아는 전문 번역가로서 번역해 주세요." |
| **② 용어집** (Glossary)                   | 원문 용어별로 한글 번역 / 음차 / 원문 유지 규칙을 표로 정의합니다. (예: Mainnet → 메인넷) |
| **③ 스타일 가이드** (Style Guide)         | 존댓말·반말, 음차 정책, 약어 처리 방식을 정의하고 어색한 번역투를 지양합니다.             |
| **④ 로케일** (Locale)                     | 날짜·시간·숫자·통화·단위를 한국 표기 규칙에 맞춰 정의합니다.                              |
| **⑤ 예시 학습** (Few-shot Examples)       | "이 문장은 이렇게 번역한다"는 실제 번역 예시를 1~3개 제공하여 결과 품질을 높입니다.       |
| **⑥ 근거 자료 조사** (Grounding) _(선택)_ | 기사화된 실제 사례를 참고 자료로 제공하여 오역 여부를 검수합니다.                         |

> 공식적으로 게시된 포스팅을 축적하여 **Mantle KR의 톤앤매너(핏)** 를 손쉽게 반영할 수 있도록 하겠습니다.

## 8. 소셜미디어 업로드 _(자동화 선택)_

2차 검수까지 완료된 글은 각 채널에 업로드하게 됩니다. 이 단계는 **자동화와 매뉴얼 모두 가능**하며, 매뉴얼 작업 역시 리소스가 크게 들지 않으므로 상황에 맞게 **선택하여 진행하실 수 있습니다**.

| 채널               | 방식   | 비고                                                                                                              |
| ------------------ | ------ | ----------------------------------------------------------------------------------------------------------------- |
| **8-1. 트윗**      | 매뉴얼 | 자동 업로드 시 계정 밴(정지) 위험이 있어 우선 매뉴얼로 진행                                                       |
| **8-2. 텔레그램**  | 자동화 | 어나운스먼트 / KOL. Google Sheet 등에서 맨틀 그룹·KOL 등의 수신처 정보를 불러와 대상별 자동 전송을 구성할 수 있음 |
| **8-3. 오픈 카톡** | 매뉴얼 | API 미제공 → 매뉴얼 업로드가 가장 간단                                                                            |
| **8-4. PR 메일**   | 자동화 | 미디어·매체 대상 보도자료 메일 발송                                                                               |

## 9. 엑셀(스프레드시트) 자동화

**Excel MCP**를 활용하여 Google Sheet를 자동화의 데이터 허브로 사용합니다. 시트는 세 가지 역할을 합니다.

**① 수신처·배포 정보 관리 (입력)**

- 맨틀 텔레그램 그룹, KOL 리스트, PR 매체 등 **배포 대상 정보를 시트로 관리**합니다.
- 8-2(텔레그램) 등 자동 전송 단계에서 이 시트를 참조하여 **대상별로 자동 발송**합니다.
- 수신처 추가·수정은 시트만 편집하면 되므로, 코드 수정 없이 KR 팀이 직접 관리할 수 있습니다.

**② 업로드 이력 기록 (출력)**

- 업로드까지 완료된 건에 대하여 **게시 일시, 채널, 링크, 상태** 등을 시트에 자동으로 기록합니다.
- 이를 통해 발행 현황을 한눈에 추적하고, 누락·중복 없이 관리할 수 있습니다.

**③ 성과(임프레션) 수집 (출력)**

- 게시된 글이 각 방에서 얼마나 노출되었는지 **임프레션(조회수) 데이터를 수집하여 시트에 기록**할 수 있습니다.
- 이를 통해 채널·글 유형별 반응을 정량적으로 파악하고, 이후 콘텐츠 전략에 활용할 수 있습니다.
- **단, 임프레션 수집은 해당 방(그룹)에 계정이 참여해 있어야 가능**합니다. (미참여 방은 수집 대상에서 제외)

> 정리하면, Google Sheet 하나로 **"누구에게 보낼지(수신처)"**, **"무엇을 언제 보냈는지(이력)"**, **"얼마나 노출됐는지(성과)"** 를 모두 관리하여, 자동화 전 과정을 사람이 손쉽게 통제할 수 있도록 합니다.

## 10. 봇 활용

앞선 자동화를 실제로 실행하는 창구로 **봇(Bot)** 을 활용할 수 있습니다. 채널별로 역할을 나누어 운영합니다.

### 10-1. 텔레그램 봇

맨틀 KR 텔레그램 방에서 콘텐츠 발행과 반응 형성을 자동으로 수행합니다.

| 기능            | 설명                                                               |
| --------------- | ------------------------------------------------------------------ |
| **자동 게시**   | 2차 검수를 통과한 글을 지정된 방에 자동으로 업로드                 |
| **자동 댓글**   | 게시글에 추가 설명·링크·이벤트 안내 등을 댓글(스레드)로 자동 등록  |
| **자동 리액션** | 게시글에 좋아요(👍) 등 리액션을 자동으로 적용하여 초기 반응을 형성 |

> ⚠️ 자동 리액션·댓글은 과도하게 사용할 경우 어뷰징으로 간주될 수 있으므로, **빈도·범위에 대한 규칙을 사전에 정하여** 운영합니다.

**📌 레퍼런스 — 고매봇**

- 텔레그램 커뮤니티 운영 봇의 대표 사례로, 자동 공지·리액션·이벤트 관리 등을 구현한 봇입니다.
- 위와 같은 텔레그램 봇 기능의 참고 모델로 삼습니다.

### 10-2. Lark 봇

Lark 봇은 **KR 팀 내부 운영(워크플로우·모니터링)** 을 담당합니다. 대외 발행용인 텔레그램 봇과 역할이 구분됩니다.

| 기능            | 설명                                                                       |
| --------------- | -------------------------------------------------------------------------- |
| **검수 알림**   | 1·2차 검수 대기 건이 생기면 담당자에게 Lark 메시지로 알림 (승인 링크 포함) |
| **승인 처리**   | Lark 메시지의 버튼으로 검수 **승인/반려**를 바로 처리 (대시보드 대체 가능) |
| **현황 리포트** | 일·주 단위로 발행 건수, 채널별 임프레션 등 성과 요약을 방으로 자동 전송    |
| **오류 알림**   | 수집·번역·업로드 과정에서 오류 발생 시 즉시 담당자에게 통보                |

> 요약하면, **텔레그램 봇 = 대외 발행/반응**, **Lark 봇 = 내부 검수/모니터링**으로 역할을 분리하여, 운영 효율과 안정성을 함께 확보합니다.
